# SurgMentor Mission

## Goal

Transform SurgMentor from a RAG-based surgical education application into a competition-grade agentic system aligned with the Kaggle AI Agents Intensive course.

## Track

Agents for Good

## Problem

Medical students and surgical residents need structured, interactive, case-based surgical training.

## Solution

SurgMentor provides:

- Surgical case retrieval
- OSCE simulation
- Clinical reasoning support
- Performance evaluation
- Personalized study plans

## Current Features

- DeepSeek API
- ChromaDB
- Jina Embeddings
- Surgical RAG
- OSCE Simulation
- Scoring Engine

## Current Limitations

- No ADK architecture
- No MCP integration
- No explicit skill system
- No agent orchestration layer
- Limited evaluation architecture

## Target Architecture

### Agent Controller

Coordinates all workflows.

### Skills

- Case Retrieval Skill
- OSCE Examiner Skill
- Clinical Reasoning Skill
- Evaluation Skill
- Study Planner Skill

### Security Layer

Guardrails and safety policies.

### Evaluation Layer

Measures quality and performance.

### Deployable Demo

Publicly accessible demonstration.

## Competition Objectives

Must clearly demonstrate:

- Agent Design
- Security
- Deployability

Optional:

- MCP
- Multi-Agent Design

## Success Criteria

A project that is educationally valuable, technically sound, publicly reproducible, and competitive for the Kaggle AI Agents Intensive Capstone Project.
